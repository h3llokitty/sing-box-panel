## Установка на чистый серверПредварительно нужен уже поднятый сервер B (второй sing-box с Hysteria2-инбаундом) —он готовится отдельно, `install.sh` настраивает только сервер A.```bashgit clone git@github.com:h3llokitty/sing-box-panel.gitcd sing-box-panelsudo bash install.sh```Скрипт интерактивно спросит:- IP и домен этого сервера- email для Let's Encrypt- домен, порт и пароль сервера B- сайт для маскировки VLESS+Reality (см. ниже)Установка включает компиляцию sing-box из исходников с поддержкой `with_v2ray_api`(нужна для статистики трафика по клиентам) — это занимает 5-15 минут.## Выбор сайта для RealityНужен реальный сайт с поддержкой TLS 1.3. Предпочтительны крупные глобальныесервисы (microsoft.com, apple.com, cloudflare.com) — их дорого блокироватьизбирательно, в отличие от локальных сервисов.Проверка кандидата:```bashopenssl s_client -connect ДОМЕН:443 -servername ДОМЕН -tls1_3 </dev/null 2>&1 | grep -E "Protocol|subject|issuer"```Должен вернуться `Protocol: TLSv1.3` и валидный сертификат.## Управление после установки```bash/root/sb-panel```Меню:1. создать клиента2. редактировать клиента (переименование, смена транспорта)3. отозвать клиента4. показать клиента (профиль, QR, URL)5. сервис (логи обращений, статистика трафика, пересборка конфига)## Требования к серверу BОтдельный sing-box с Hysteria2-инбаундом, слушающий на известном порту сизвестным паролем — эти данные вводятся при установке сервера A.## Структура репозиторияinstall.sh              — установщик для чистого сервераvpn-setup.sh             — CLI-менеджер клиентов (запускается через /root/sb-panel)config.env.example       — справочный список параметров (install.sh спрашивает их сам)templates/template.json           — клиентский профиль под sing-box 1.12+ (modern)template-legacy.json     — клиентский профиль под sing-box 1.11.x (legacy)stats.proto              — protobuf-описание для сбора статистики трафика
